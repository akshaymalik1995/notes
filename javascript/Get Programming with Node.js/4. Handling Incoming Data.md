Every time a user visits a URL that leads to your application, a request is made, and each request must be processed by the code you write.

In this lesson, you learn how to gather and process some of the information in these requests. You also build application routes—code logic to match requests with appropriate responses.

In your server code, you have a server object that has a callback function, (req, res) ⇒ {}, which is run every time a request is made to the server. With your server running, if you visit localhost:3000 in your browser and refresh the page, that callback function is run twice—once on every refresh.

In other words, upon receiving a request, the server passes a request and response object to a function where you can run your code. 

**Another way to create a server**

```js
//index.js

// create server and store it in a variable

const app = http.createServer()

// On receiving request

app.on("request", (request, response) => {
    console.log("We have received an HTTP request")
    // Set Headers
    response.writeHead(statusCode.OK, {
        'Content-Type' : 'text/html'
    })
    // Create response 
    const responseMessage = `<h1> This is a response from the server </h2>`
    // Send response
    response.write(responseMessage)
    // End response
    response.end()
})
```

The request object, like most objects in Node.js, can also listen for events, similarly to the server. If someone makes a POST request to the server (trying to send data to the server), the content of that POST lives in the request’s body. Because a server never knows how much data is being sent, posted data comes into the http server via chunks.

> Data chunks allow information to stream into and out of a server. Instead of waiting for a large set of information to arrive at the server, Node.js allows you to work with parts of that information as it arrives via the ReadableStream library.

To collect all the posted data with a server, you need to listen for each piece of data received and arrange the data yourself. Luckily, the request listens for a specific dataevent. `request.on("data")` is triggered when data is received for a specific request. You need to define a new array, body, outside this event handler and sequentially add the data chunks to it as they arrive at the server. When all the data chunks are received, they can be collected as a single data item.

### Receiving The Data

Within the `app.on("request")` code block, add the new request event handlers to read incoming data. Every time a request is made to the server, 
you execute the code in the callback function. An array is created and referred to as body, and every time data from the request is received, you process it in another callback function. The received data is added to the body array. When the transmission of data is complete, you execute code in a third callback function. The body array is turned into a String of text, and the request’s contents are logged to your console.


```js
app.on("request", (request, response) => {
    ////////////////////////////
    ////REQUEST////////////////

    //Create an array where we will add the data received
    let body = []
    // Listen for the data
    request.on("data", chunk => {
        console.log("We have recieved a data chunk")
        console.log(chunk)
        body.push(chunk)
    })

    // Listen for the end of the data

    request.on("end", () => {
        console.log("The post request has ended")
        // Convert the body array into string
        body = Buffer.concat(body).toString()
        console.log(`Received Body Contents: ${body}`)
    })

    //////////////////////////////
    /////RESPONSE /////////////
    
    
    console.log("We have received an HTTP request")
    // Set Headers
    response.writeHead(statusCode.OK, {
        'Content-Type' : 'text/html'
    })
    // Create response 
    const responseMessage = `<h1> This is a response from the server. You have opened the url localhost:3000${request.url} </h2>`
    // Send response
    response.write(responseMessage)
    // End response
    response.end()
})



```

With this added code, your application is prepared to receive posted data collected into an array and converted back to String format. When an event is triggered, indicating that some chunk of data reached the server, you handle that data by adding the chunk (represented as a Buffer object) to an array. When the event indicating the request’s connected has ended, you follow up by taking all the array’s contents and turn them into text you can read. To test this process, try sending a POST request to your server from terminal.

### Adding Routes to a web application

A route is a way of determining how an application should respond to a request made to a specific URL. An application should route a request to the home page differently from a request to submit login information.

```js
const server = http.createServer()

// Create response routes

const routes = {
    "/" : "<p>Welcome to the home page</p>",
    "/login" : "<p>Welcome to the login page</p>",
    "/register" : "<p>Welcome to the register page</p>",
}


// Create  request listener

server.on("request", (request, response) => {
   

    //////////////////////////////
    /////RESPONSE /////////////
    
    
    console.log(`We have received an HTTP request from localhost:${port}${request.url}`)
    // Set Headers
    response.writeHead(statusCode.OK, {
        'Content-Type' : 'text/html'
    })

    // Check url and send response accordingly

    if (routes[request.url]) {
        response.write(routes[request.url])
    }
    else {
        response.write("<p>This page does not exist</p>")
    }
    
    
    // End response
    response.end()
})
```
